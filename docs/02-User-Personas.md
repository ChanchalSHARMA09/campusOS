# User Personas - CampusOS

## Overview

This document defines the primary user personas for CampusOS. These personas represent the different types of users who will interact with the platform and help guide feature development and design decisions.

---

# Persona 1: Student

## Basic Information

- **Role:** Student
- **Age:** 18–25 years
- **Technical Knowledge:** Beginner to Intermediate
- **Primary Device:** Laptop / Smartphone

## Goals

- View attendance and academic performance.
- Submit assignments online.
- Download lecture notes and study materials.
- Apply for internships and placements.
- Build resumes for campus recruitment.
- Participate in clubs and campus events.
- Communicate with classmates and faculty.

## Pain Points

- Multiple portals for different services.
- Missing assignment deadlines.
- Difficulty accessing study materials.
- Lack of centralized placement resources.
- Delayed communication with faculty.

## Needs

- Simple and user-friendly dashboard.
- Quick access to academic resources.
- Real-time notifications.
- Secure authentication.
- Mobile-friendly interface.

---

# Persona 2: Faculty

## Basic Information

- **Role:** Faculty Member
- **Age:** 28–60 years
- **Technical Knowledge:** Intermediate
- **Primary Device:** Laptop

## Goals

- Mark student attendance.
- Upload assignments and notes.
- Evaluate student submissions.
- Communicate with students.
- Track student performance.

## Pain Points

- Manual attendance process.
- Managing assignments through multiple platforms.
- Difficulty tracking submissions.
- Time-consuming administrative tasks.

## Needs

- Easy attendance management.
- Assignment evaluation tools.
- Organized student records.
- Fast communication system.

# Persona 3: Placement Officer

## Basic Information

- **Role:** Placement Officer
- **Age:** 30–55 years
- **Technical Knowledge:** Intermediate
- **Primary Device:** Desktop / Laptop

## Goals

- Publish internships and job opportunities.
- Manage campus recruitment drives.
- Track student applications.
- Analyze placement statistics.
- Communicate placement updates.

## Pain Points

- Manual student record management.
- Difficulty sharing job updates.
- Lack of centralized placement data.

## Needs

- Placement dashboard.
- Resume database.
- Student application tracking.
- Company management system.

---

# Persona 4: Club Coordinator

## Basic Information

- **Role:** Club Coordinator
- **Age:** 19–24 years
- **Technical Knowledge:** Intermediate
- **Primary Device:** Laptop / Smartphone

## Goals

- Create and manage clubs.
- Organize campus events.
- Publish announcements.
- Manage club members.

## Pain Points

- Difficulty informing members.
- Manual event registration.
- Poor event visibility.

## Needs

- Club management dashboard.
- Event registration system.
- Notification system.
- Member management tools.

---

# Persona 5: Hostel Administrator

## Basic Information

- **Role:** Hostel Administrator
- **Age:** 30–55 years
- **Technical Knowledge:** Basic to Intermediate
- **Primary Device:** Desktop

## Goals

- Allocate hostel rooms.
- Manage complaints.
- Approve leave requests.
- Maintain hostel records.

## Pain Points

- Paper-based records.
- Manual complaint handling.
- Difficult room allocation process.

## Needs

- Digital hostel management.
- Complaint tracking.
- Leave approval system.
- Student database.

---

# Persona 6: System Administrator

## Basic Information

- **Role:** Administrator
- **Age:** 25–50 years
- **Technical Knowledge:** Advanced
- **Primary Device:** Desktop

## Goals

- Manage users and roles.
- Monitor system performance.
- Maintain platform security.
- Manage permissions.
- Configure platform settings.

## Pain Points

- Managing a large number of users.
- Security monitoring.
- Manual user verification.
- System maintenance.

## Needs

- Admin dashboard.
- User management tools.
- Role-based access control.
- Activity logs.
- Analytics dashboard.

---

# Summary

| Persona | Main Goal | Primary Features |
|----------|-----------|------------------|
| Student | Manage academics and placements | Dashboard, Assignments, Attendance, Resume Builder |
| Faculty | Manage teaching activities | Attendance, Assignments, Notes |
| Placement Officer | Manage recruitment | Jobs, Applications, Analytics |
| Club Coordinator | Manage clubs and events | Clubs, Events, Announcements |
| Hostel Administrator | Manage hostel operations | Hostel, Complaints, Leave Requests |
| System Administrator | Manage entire platform | User Management, Security, Analytics |
